Gradient Descent

Gradient Descent iteratively minimises a cost function by moving in the direction of the steepest descent. Adjust the learning rate \(\eta\) to see how step size affects convergence on \(J(\theta) = (\theta - 2)^2\).

Core Equations

MSE Cost Function $$J(\theta) = \frac{1}{2m}\sum_{i=1}^{m}\left(\hat{y}^{(i)} - y^{(i)}\right)^2$$
Gradient Descent Update $$\theta_j \leftarrow \theta_j - \eta \frac{\partial J}{\partial \theta_j}$$
Batch GD (vectorised) $$\boldsymbol{\theta} \leftarrow \boldsymbol{\theta} - \frac{\eta}{m}\mathbf{X}^T(\mathbf{X}\boldsymbol{\theta} - \mathbf{y})$$
Normal Equation (closed-form) $$\hat{\boldsymbol{\theta}} = \left(\mathbf{X}^T\mathbf{X}\right)^{-1}\mathbf{X}^T\mathbf{y}$$
Step: 0 θ: — J(θ): —
Insight: For \(J(\theta) = (\theta-2)^2\), the gradient is \(2(\theta-2)\). Convergence is guaranteed when \(\eta < 1\). At \(\eta = 1\) the algorithm oscillates; above \(\eta = 1\) it diverges.

Polynomial Regression

By adding polynomial features \(x^2, x^3, \ldots\) to linear regression we can model non-linear data. Too few terms → underfitting; too many → overfitting. Adjust the degree slider to see the effect.

Core Equations

Polynomial Feature Expansion $$\hat{y} = \theta_0 + \theta_1 x + \theta_2 x^2 + \cdots + \theta_d x^d$$
Normal Equation (closed-form fit) $$\boldsymbol{\theta} = (\mathbf{X}^T\mathbf{X})^{-1}\mathbf{X}^T\mathbf{y}$$
Training MSE $$\text{MSE}_\text{train} = \frac{1}{m}\sum_{i=1}^{m}\left(\hat{y}^{(i)} - y^{(i)}\right)^2$$
Train MSE: — Degree: 1
Insight: The true underlying function is \(\sin(x) + \epsilon\) (noise \(\epsilon\)). A degree-1 line underfits. A degree ≈ 3–5 fits well. Higher degrees memorise the noise — overfitting. The Normal Equation solves \((\mathbf{X}^T\mathbf{X})^{-1}\mathbf{X}^T\mathbf{y}\) exactly.

Regularisation: Ridge vs Lasso vs Elastic Net

Regularisation constrains model weights to reduce overfitting. Increase \(\alpha\) to strengthen the penalty and watch how each method handles the coefficients differently.

Cost Functions

Ridge (\(\ell_2\) penalty) $$J(\boldsymbol{\theta}) = \text{MSE}(\boldsymbol{\theta}) + \alpha\sum_{j=1}^{n}\theta_j^2$$
Lasso (\(\ell_1\) penalty) $$J(\boldsymbol{\theta}) = \text{MSE}(\boldsymbol{\theta}) + \alpha\sum_{j=1}^{n}|\theta_j|$$
Elastic Net $$J(\boldsymbol{\theta}) = \text{MSE}(\boldsymbol{\theta}) + r\alpha\!\sum|\theta_j| + \tfrac{1-r}{2}\alpha\!\sum\theta_j^2$$
Ridge shrinks all coefficients toward zero but keeps them non-zero. Lasso applies soft-thresholding — some coefficients become exactly zero, performing automatic feature selection.

Classification: Sigmoid & Softmax

Understand supervised vs unsupervised learning, and how sigmoid (N=2) and softmax (N>2) turn raw scores into predictions.

Supervised Learning

Key Definitions

Supervised Learning $$\text{Given } \{(\mathbf{x}^{(i)}, y^{(i)})\}_{i=1}^{m}, \text{ learn } f: \mathbf{x} \to y$$
Unsupervised Learning $$\text{Given } \{\mathbf{x}^{(i)}\}_{i=1}^{m} \text{ (no labels), find structure}$$
Supervised examples:
  • Classification: spam vs not-spam
  • Regression: predict house price from features
  • Object detection: label objects in images
Unsupervised examples:
  • Clustering: group customers by behaviour
  • Dimensionality reduction: PCA, t-SNE
  • Anomaly detection: find unusual patterns

Sigmoid — N = 2 Classes

Sigmoid function $$\sigma(z) = \frac{1}{1 + e^{-z}}$$
Logit from model $$z = \mathbf{w}^T\mathbf{x} + b$$
Decision rule $$\hat{y} = \begin{cases}1 & \text{if } \sigma(z) \geq 0.5 \\ 0 & \text{otherwise}\end{cases}$$
σ(z) =
Why sigmoid for N=2? It maps any real logit to (0,1), which we interpret as P(class=1). The threshold at σ(z)=0.5 is equivalent to z=0, i.e. the decision boundary where wTx + b = 0. At z→+∞, σ→1 (certain positive); at z→−∞, σ→0 (certain negative).

Softmax — N > 2 Classes

Softmax function $$\text{softmax}(\mathbf{z})_k = \frac{e^{z_k}}{\displaystyle\sum_{j=1}^{K} e^{z_j}}$$
Key property $$\sum_{k=1}^{K} \text{softmax}(\mathbf{z})_k = 1$$
Decision $$\hat{y} = \arg\max_k\; \text{softmax}(\mathbf{z})_k$$
Why softmax for N>2? We need K outputs that are (1) all positive and (2) sum to 1. Exponentiation ensures positivity; dividing by the sum normalises. Notice: if K=2, softmax reduces to sigmoid for class 1.

Feature Scaling

See how standardisation and min-max scaling transform features, and why unscaled features cause gradient descent to converge slowly.

Transformation table

PointOriginal xScaled x'

Scaling Formulas

Standardisation (Z-score) $$x' = \frac{x - \mu}{\sigma}$$ Result: mean=0, std=1
Min-Max Normalisation $$x' = \frac{x - x_{\min}}{x_{\max} - x_{\min}}$$ Result: range [0, 1]
When does scaling matter? Any algorithm that uses distances or gradient descent is sensitive to feature scale: k-NN, SVM, gradient descent, neural networks. The Normal Equation is scale-invariant (but scaling still helps numerically). Decision trees are scale-invariant.
Left: Unscaled — one feature varies 0–1000, the other 0–1. The loss surface is a very stretched ellipse; GD makes tiny steps in the flat direction while overshooting in the steep direction.
Right: After scaling — both features standardised. Circular contours let GD converge in far fewer steps on a straight path.

Why Shape Matters

Unscaled loss contour $$J \propto \theta_1^2 + (50\,\theta_2)^2 \quad \text{(elongated)}$$
After standardisation $$J \propto \theta_1^2 + \theta_2^2 \quad \text{(circular)}$$
GD update rule $$\boldsymbol{\theta} \leftarrow \boldsymbol{\theta} - \eta\,\nabla_{\boldsymbol{\theta}} J$$
With circular contours, the negative gradient always points toward the minimum. With elongated contours, the gradient points mostly across the narrow direction — causing the characteristic zigzag pattern and slow convergence.

Calculus

Derivatives, differentiation rules, and partial derivatives — the tools behind gradient descent and the Normal Equation.

Derivative Definition

Limit definition $$f'(a) = \lim_{h \to 0} \frac{f(a+h) - f(a)}{h}$$
Geometric meaning $$\text{slope of tangent line at } x = a$$
Try this: Start with h = 1 (secant line, yellow dashes), then click h ÷ 10 repeatedly. Watch the secant approach the tangent (green line) and see the error shrink toward zero. This is the limit definition of the derivative.

Select a Rule

All Rules Summary

Sum / Difference $$(f \pm g)' = f' \pm g'$$
Constant multiple $$(cf)' = cf'$$
Power $$(x^n)' = nx^{n-1}$$
Product $$(fg)' = f'g + fg'$$
Chain $$[f(g(x))]' = f'(g(x)) \cdot g'(x)$$
In ML: The gradient of the RSS cost function uses the chain rule: $$\frac{\partial \text{RSS}}{\partial \boldsymbol{\theta}} = -2X^T(\mathbf{y} - X\boldsymbol{\theta})$$ This is derived by applying the chain rule to \(\|\mathbf{y} - X\boldsymbol{\theta}\|^2\).

Partial Derivatives

Definition $$\frac{\partial f}{\partial x} = \lim_{h\to 0}\frac{f(x+h,y)-f(x,y)}{h}$$
Gradient vector $$\nabla_{\mathbf{x}} f = \begin{bmatrix}\partial f/\partial x_1 \\ \partial f/\partial x_2\end{bmatrix}$$
Key idea: A partial derivative ∂f/∂x treats y as a constant — it measures the slope in only the x-direction. The gradient ∇f points in the direction of steepest ascent. Gradient descent moves in the opposite direction.

Ridge & Lasso Regularisation — Geometry

Visualise why Ridge (ℓ₂) shrinks weights toward zero but rarely eliminates them, while Lasso (ℓ₁) tends to produce sparse solutions by hitting the corners of its diamond constraint region.

What you're seeing

Blue ellipses MSE loss contours — ellipses centred at the OLS (unconstrained) optimum ★
Coloured region Constraint region: circle (Ridge) or diamond (Lasso). The regularised solution must lie inside.
Coloured dot Regularised optimum — where the smallest possible loss ellipse first touches the constraint region.

Regularisation Cost Functions

Ridge Regression (ℓ₂) $$J(\boldsymbol{w}) = \text{MSE}(\boldsymbol{w}) + \alpha \sum_{j=1}^{n} w_j^2$$

Penalises the sum of squared weights. Equivalent to constraining \(\|\boldsymbol{w}\|_2^2 \leq t\). Geometric constraint region: a ball (circle in 2D).

Lasso Regression (ℓ₁) $$J(\boldsymbol{w}) = \text{MSE}(\boldsymbol{w}) + \alpha \sum_{j=1}^{n} |w_j|$$

Penalises the sum of absolute weights. Equivalent to constraining \(\|\boldsymbol{w}\|_1 \leq t\). Geometric constraint region: an ℓ₁ ball (diamond in 2D).

Elastic Net (combined) $$J(\boldsymbol{w}) = \text{MSE}(\boldsymbol{w}) + r\alpha\sum|w_j| + \frac{1-r}{2}\alpha\sum w_j^2$$

Mix ratio \(r\) between Ridge and Lasso. Best of both worlds — sparse and stable.

Why Lasso sets weights to zero: The ℓ₁ diamond has sharp corners on the axes. The MSE loss ellipses (which are smooth) tend to first touch the diamond at a corner — a point where one weight is exactly zero. The ℓ₂ circle is smooth everywhere, so the tangent point is almost never at an axis, and weights are merely shrunk toward zero.
Closed-form solutions: Ridge has an exact solution \(\hat{\boldsymbol{\theta}} = \left(\mathbf{X}^T\mathbf{X} + \alpha\mathbf{I}\right)^{-1}\mathbf{X}^T\mathbf{y}\). Lasso does not — it requires an iterative algorithm (coordinate descent / sub-gradient methods) because the ℓ₁ penalty is not differentiable at zero.

K-Nearest Neighbours (KNN)

KNN is an instance-based classifier — it memorises training data and classifies a query point by majority vote among its k nearest neighbours. Click anywhere on the canvas to place a query point.

Click the canvas to place a query point.

Key Equations

Minkowski Distance (generalised) $$d_p(\mathbf{x},\mathbf{z}) = \left(\sum_{j=1}^{n}|x_j - z_j|^p\right)^{1/p}$$

\(p=2\): Euclidean · \(p=1\): Manhattan · \(p\to\infty\): Chebyshev

Majority Vote (classification) $$\hat{y} = \underset{c}{\arg\max}\; \sum_{i \in \mathcal{N}_k(\mathbf{x})} \mathbf{1}[y^{(i)} = c]$$
Bias–Variance tradeoff: Small k → low bias, high variance (complex, jagged boundary, overfits noise). Large k → high bias, low variance (smooth boundary, may underfit).

Important: KNN requires feature normalisation — features on larger scales dominate the distance calculation and drown out smaller-scale features.

Tom Mitchell's Definition (1997)

"A computer program is said to learn from experience E with respect to some class of tasks T and performance measure P, if its performance at tasks in T, as measured by P, improves with experience E."

— Tom Mitchell, Machine Learning, 1997

E
Experience

The training data the model is exposed to. More experience generally leads to better performance (up to a point).

T
Task

What the model should do — classify images, predict prices, play a game, translate text, etc.

P
Performance

How we measure success at the task — accuracy, MSE, F1 score, win rate, BLEU score, etc.
